# 投资公司发展物语 V2.0 设计文档 v0.5

> **版本说明**  
> - 本文档为 **v0.5** 完整设计，合并实现规划与规则速查。
> - 继承 **v0.4** 全部功能（股份、上市、NPC、增发、自交易）。
> - 全篇与仓库 **`investment-sim/`** 对齐（`SCHEMA_VERSION` 待升级）；与代码冲突时以代码为准。
> - 新增四大业务线：**其他公司系统**、**房地产投资**、**初创公司投资**。
> - **并购系统**暂不实现（v0.6规划）。

---

## 0. v0.5 实现约定与变更摘要

| 主题 | v0.5 约定 |
| --- | --- |
| **其他公司系统** | 新增NPC公司生态，支持页签分类浏览（投资机构/上市公司/国际巨头/私有企业）。 |
| **页签分类** | "其他公司"按钮打开面板，内含：投资机构、上市公司、国际巨头、私有企业四个页签。 |
| **房地产投资** | 扩张期解锁，12-48个月长周期项目，可烂尾（无收益）。 |
| **初创公司投资** | 扩张期解锁，多轮投资（种子→天使→A→B→C→退出），失败直接消失。 |
| **在研项目** | 每家初创公司自带行业相关的在研项目（如"高等级芯片"、"淘宝网"）。 |
| **解锁延续** | 新功能沿用v0.3阶段系统：扩张期解锁房地产和初创投资。 |
| **业务组系统** | 扩张期解锁，玩家可创建业务组并派遣母公司员工；通过研发/扩展/专利三条主线成长，可IPO上市生成可交易股票。 |
| **并购系统** | **v0.5暂不实现**，留待v0.6。 |

### 目录结构约定

> **⚠️ 重要**：本文档中所有路径均以**仓库根目录**为基准：
> - 代码路径：`investment-sim/`（HTML/JS 主应用）
> - 数据路径：`data/investment-sim/`（JSON 数据文件，与代码目录同级）

| 类型 | 实际目录 | 说明 |
|------|----------|------|
| 前端代码 | `investment-sim/js/core/` | ES Module 核心逻辑 |
| 静态数据 | `data/investment-sim/` | JSON 配置文件，被前端 fetch 加载 |
| 入口页面 | `investment-sim/index.html` | 游戏主入口 |

---

## 1. 实现策略与子版本

| 子版本 | 目标 | 核心交付 | 验收要点 |
|--------|------|----------|----------|
| **v0.5.0** | 其他公司系统 | NPC公司数据、页签UI、基础关系系统 | 可查看四类公司，有基础信息展示 |
| **v0.5.1** | 房地产投资 | 项目池、投资流程、建设期结算、烂尾机制 | 可开展投资，烂尾无收益 |
| **v0.5.2** | 初创公司投资 | 初创池、多轮投资、在研项目、退出机制 | 可投资初创，失败消失 |
| **v0.5.3** | 业务组系统 | 业务组创建、员工派遣、月度tick、研发/扩展/专利、IPO上市 | 可创建业务组，完成完整生命周期至IPO |

|**代码主路径**（相对于 `investment-sim/`）：
- `js/core/otherCompanies.js` — 其他公司系统
- `js/core/realEstate.js` — 房地产投资
- `js/core/startupInvest.js` — 初创公司投资
- `js/core/businessGroups.js` — 业务组系统

|**数据主路径**（相对于仓库根目录）：
|- `other-companies.json` — NPC公司定义
|- `real-estate-projects.json` — 房地产项目模板
|- `startup-projects.json` — 8行业×20个项目名
|- `startup-invest.json` — 融资轮次配置
|- `stocks-futures.json` — 股票期货基础数据
|- `business-groups.json` — 业务组持久化数据
|- `employees.json` — 员工池（与母公司员工复用）

## 6. 核心流程

### 6.1 房地产投资项目生命周期

```
项目发现
    ↓
尽职调查（1月，费用=投资额×1%，可放弃，损失尽调费）
    ↓（尽调通过）
土地获取（2月，费用=投资额×30%，不可退出）
    ↓
建设施工（8月，费用=投资额×40%，按月投入）
    ↓（每月判定：烂尾率检查）
销售回款（6月，费用=投资额×20%，按月回收）
    ↓
项目结算（1月，费用=投资额×9%，结算ROI）
    ↓
[完成] 收益入账 / [烂尾] 无收益
```

**烂尾判定**：
- 建设期每月进行一次随机判定
- 基础烂尾率：住宅5%、商业10%、写字楼12%、产业园20%
- 员工执行力影响：每点执行力降低0.5%烂尾率
- 烂尾后项目终止，已投入资金损失

### 6.2 初创公司投资生命周期

```
项目发现（每月随机生成3-5个BP）
    ↓
尽职调查（1-2月，费用=拟投资额×2%）
    ↓
投资谈判（估值、比例、条款）
    ↓
投资完成（资金划出，获得股权）
    ↓
投后管理（每6月检查一次）
    ├── 进入下一轮融资（估值增长，可跟投）
    ├── 被收购（获得收购溢价）
    ├── IPO（转为上市公司股权）
    ├── 破产清算（投资归零，公司消失）
    └── 维持现状（继续等待）
```

**退出计算**：
- 被收购：估值×1.5-3倍
- IPO：估值×2-5倍
- 下一轮融资：新估值×持股比例

## 11. 业务组系统（已实装 v0.5.3+）

> **状态**：已实装。v0.5.3 对业务组玩法进行了重构：从「月度资金分配」改为「按钮式即时操作」，并引入产品化研发模型。
> - 代码与本文档冲突时以代码为准。

业务组（Business Group）是玩家派遣母公司员工、注资成立的独立经营实体。通过**研发产品**、**扩展市场**、**研发专利**、**人才培养**四条线驱动成长，最终可申请 IPO 上市并生成可交易股票。

- 货币单位：万。
- 目标定位：偏策略，按钮式即时操作，产品进度可视化。

### 11.1 创建条件与流程

**解锁阶段**：扩张期。

**创建入口**：「新开业务」→「业务组」。

**创建参数**：

| 参数 | 规则 |
|------|------|
| 名称 | 玩家自定义 |
| 行业 | 8 个行业下拉选择 |
| 负责人 | 母公司**空闲员工**（无在营业务、不在其他业务组） |
| 初始资金 | 最低 **100 万**，从公司现金扣除 |
| 月烧钱率 | 已废弃（不再使用），保留字段兼容旧存档 |

**创建后自动生成**：
- 一个在研产品（根据行业从 `startup-projects.json` 中选取名称）
- `stage = 'formation'`
- `metrics.valuationWan = fundingWan × 0.6`

### 11.2 团队管理

**人数上限**：最多 **5 人**。

| 操作 | 条件 | 效果 |
|------|------|------|
| **调入员工** | 空闲、不在其他组、团队未满 | 加入 `teamIds` |
| **移除员工** | 详情页点击移除 | 从 `teamIds` 移除；移除负责人则清空 `leaderId` |
| **自动裁撤** | `teamIds.length === 0` | 次月 tick 时裁撤，资金返还母公司 |

### 11.3 即时操作按钮（按钮式，非月度分配）

业务组详情页提供 4 个操作按钮，点击立即执行：

| 按钮 | 花费 | 效果 | 限制 |
|------|------|------|------|
| **🔬 研发产品** | 50 万 | 生成新的在研产品 | 已有在研产品时不可点击 |
| **📈 扩展业务** | 50 万 | 市占率直接 +0.5% | 资金不足或已达上限时不可点击 |
| **📋 专利申请** | — | 占位，暂无效果 | 提示「功能开发中」 |
| **🎓 人才培养** | — | 占位，暂无效果 | 提示「功能开发中」 |

> **设计变更**：旧版的「月度投入设置」（rdWan/expandWan/patentWan 输入框）已移除，改为按钮式即时消费。

### 11.4 在研产品系统

**产品生成**：
- 创建业务组时自动生成 1 个在研产品。
- 点击「研发产品」按钮生成新产品（需当前无在研产品）。
- 产品名根据行业从 `startup-projects.json` 中选取。

**产品进度**：
- 每月自动推进 **20%**（由 `tickMonth` 执行）。
- 进度达到 100% 时产品完成，`productLevel += 1`。
- 完成后的产品永久保留在列表中，作为已完成产品展示。

**产品列表 UI**：
- 显示产品名、进度条、状态（研发中/已完成）。
- 进度条实时反映 `progress` 字段（0~1）。

### 11.5 月度 tick 计算

每月点击「下一个月」时执行：

```
1. 扣除员工工资 = 团队人数 × 5万/人/月
2. 自动推进在研产品进度（+20%）
3. 计算收入 = marketSizeWan × marketShare × productMultiplier
4. 收入入账（增加业务组资金）
5. 更新 TTM 收入与估值
6. 检查资金耗尽 → failed
7. 状态迁移判定
```

**收入公式**：
```
productMultiplier = 1 + productLevel × 0.05 + patentValueWan / marketSizeWan
monthlyRevenueWan = marketSizeWan × marketShare × productMultiplier
```

**估值公式**：
```
valuationWan = 6 × ttmRevenueWan + patentValueWan
```

### 11.6 状态机

```
formation → incubation → growth → mature → exit(IPO) / failed
```

| 迁移 | 条件 |
|------|------|
| `formation` → `incubation` | `initialFundingWan ≥ 1000` 且 `teamIds.length ≥ 1` |
| `incubation` → `growth` | `productLevel ≥ 2` 且 `marketShare ≥ 0.005` |
| `growth` → `mature` | `marketShare ≥ 0.05` 或 `valuationWan ≥ 50000` |
| `mature` → `exit` | 玩家主动申请 IPO 且 `valuationWan ≥ 200000` |
| 任意 → `failed` | `fundingWan ≤ 0` |

### 11.7 母公司注资与裁撤

**注资**：从母公司现金划转至业务组资金，无次数限制。

**手动裁撤**：玩家主动裁撤，剩余资金返还母公司，人员释放。

### 11.8 IPO 机制

**条件**：`valuationWan ≥ 200,000`。

**流程**：点击「申请 IPO」→ 生成股票对象 → 加入 `config.stocks` → 可立即交易。

```json
{
  "stockId": "STKBG-BG001",
  "name": "杭州互联网科技股份有限公司",
  "symbol": "BG001",
  "totalSharesWan": 10000,
  "freeFloatSharesWan": 5000,
  "initialPrice": 20.0,
  "industry": "tech",
  "playerHoldingsWan": { "PLAYER-001": 5000 }
}
```

### 11.9 数值与常量速查

| 常量 | 默认值 | 说明 |
|------|--------|------|
| `maxTeamSize` | 5 | 团队人数上限 |
| `employeeSalaryWan` | 5 | 每人每月工资（万） |
| `productAutoProgressPerMonth` | 0.20 | 产品每月自动进度 |
| `rdCostPerClick` | 50 | 研发产品点击花费（万） |
| `expandCostPerClick` | 50 | 扩展业务点击花费（万） |
| `expandShareGain` | 0.005 | 扩展一次市占率提升（0.5%） |
| `marketSizeWan` | 1,000,000 | 市场规模基数 |
| `revenueMultiplier` | 6 | TTM 收入估值倍数 |
| `ipoValuationThresholdWan` | 200,000 | IPO 估值门槛 |

### 11.10 UI 总览

**列表页**：业务组卡片（名称、行业、估值、月营收、市占率、产品等级）。

**详情页**：
- **概览**：4 个核心 KPI（估值、月营收、市占率、产品等级）。
- **团队**：成员列表（`x/5` 人数显示），调入/移除。
- **在研产品**：产品列表 + 进度条。
- **即时操作**：4 个按钮（研发/扩展/专利/培养）。
- **资金**：业务组资金、月工资支出、母公司注资。
- **操作**：IPO、裁撤。

### 11.11 与主游戏的集成

**月度顺序**：
1. `bgManager.tickMonth(state)` — 业务组月度计算
2. `endTurn(state, config)` — 主月结
3. 保存到 localStorage

**存档**：`investment-sim:bg` key 独立存储业务组数据。
